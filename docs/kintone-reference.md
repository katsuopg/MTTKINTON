## 1. フォーム設計

### 1.1 フィールドタイプ一覧

#### テキスト系

| フィールド | 用途 | 主な設定項目 |
|---|---|---|
| 文字列（1行） | 1行テキスト。計算式も設定可（結果は文字列） | 必須、重複禁止（有効時最大64文字）、最小/最大文字数、初期値（ログインユーザー可） |
| 文字列（複数行） | 複数行テキスト | 必須 |
| リッチエディター | 書式付きテキスト（太字・色・リスト・テーブル・画像等） | 必須 |
| 数値 | 数値入力。桁区切り・単位記号対応 | 必須、重複禁止、最小/最大値、初期値、小数点以下桁数、単位記号 |

#### 計算

| フィールド | 用途 | 主な設定項目 |
|---|---|---|
| 計算 | 計算式による自動計算結果を表示（数値） | 計算式、表示形式（数値/時間/時刻/日付/日時）、小数点以下桁数、単位記号 |

#### 選択肢系

| フィールド | 用途 | 主な設定項目 |
|---|---|---|
| ラジオボタン | 1つだけ選択（常にいずれか選択状態） | 項目（最大1,000個、各128文字）、並び（横/縦）、初期値 |
| チェックボックス | 複数選択可能 | 項目（最大1,000個）、並び（横/縦）、初期値 |
| 複数選択 | 複数選択可能（常に縦並び） | 項目（最大1,000個）、初期値 |
| ドロップダウン | ドロップダウンから1つ選択 | 項目（最大1,000個）、初期値 |

#### 日時系

| フィールド | 用途 | 主な設定項目 |
|---|---|---|
| 日付 | 年月日（カレンダーUI） | 必須、重複禁止、初期値（登録時の日付も可） |
| 時刻 | 時刻 | 必須、重複禁止、初期値 |
| 日時 | 年月日と時刻 | 必須、重複禁止、初期値 |

#### リンク・ファイル系

| フィールド | 用途 | 主な設定項目 |
|---|---|---|
| リンク | URL、電話番号、メールアドレス | 入力値の種類、重複禁止（最大64文字）、文字数制限 |
| 添付ファイル | ファイル添付（1ファイル最大1GB、個数制限なし） | サムネイルの大きさ |

#### ユーザー選択系

| フィールド | 用途 | 主な設定項目 |
|---|---|---|
| ユーザー選択 | kintoneユーザーを選択（複数可） | 初期値（ログインユーザー可） |
| 組織選択 | 組織を選択（複数可） | 初期値（ログインユーザーの所属組織可） |
| グループ選択 | グループを選択（複数可） | 初期値（ログインユーザーの所属グループ可） |

#### アプリ間連携

| フィールド | 用途 | 主な設定項目 |
|---|---|---|
| ルックアップ | 他アプリのデータを参照・コピー取得 | 関連付けるアプリ、コピー元フィールド、ほかのフィールドのコピー、絞り込み条件、ソート |
| 関連レコード一覧 | 条件に一致するレコードを一覧表示（詳細画面のみ） | 参照するアプリ、表示条件、絞り込み、表示フィールド、ソート、最大表示件数 |

#### フォーム装飾

| フィールド | 用途 |
|---|---|
| ラベル | 見出しや説明文（書式・色・サイズ変更可） |
| スペース | 空白エリア（カスタマイズ用ボタン配置にも利用） |
| 罫線 | フィールド間の区切り線 |
| グループ | 複数フィールドをグループ化し展開/折りたたみ |
| テーブル | 入力行を動的に追加できる表形式 |

#### レコード情報（自動設定、変更不可）

| フィールド | 用途 |
|---|---|
| レコード番号 | レコード固有の採番（自動付与） |
| 作成者 | 最初に保存したユーザー |
| 作成日時 | 最初に保存した日時 |
| 更新者 | 最後に更新したユーザー |
| 更新日時 | 最後に更新した日時 |

### 1.2 フォーム設計の操作

- フィールドは**最大500個**まで配置可能
- フィールドのドラッグ&ドロップで配置
- 配置済みフィールドのフィールドタイプは変更不可
- フィールドの複製可能（フィールドコード以外コピー）
- **削除するとそのフィールドのデータもすべて削除される**
- 他アプリのルックアップ/関連レコードで参照中のフィールドは削除不可

### 1.3 テーブル

- 入力行を動的に追加できる表形式
- **テーブルに含められないフィールド**: ラベル、関連レコード一覧、スペース、罫線、グループ、テーブル、レコード番号、作成者/更新者、作成日時/更新日時
- テーブルはテーブルやグループ内には配置不可

### 1.4 自動計算

#### 演算子

| 演算子 | 説明 |
|---|---|
| `+` `-` `*` `/` | 四則演算 |
| `^` | べき算（-100〜100乗） |
| `&` | 文字列結合 |
| `=` `!=` `<>` `<` `<=` `>` `>=` | 比較演算子 |

**優先順位**: 単項`+`/`-` → `^` → `*`/`/` → `+`/`-` → `&` → 比較演算子

#### 関数

| 関数 | 説明 |
|---|---|
| `SUM(値1, 値2, ...)` | 合計（テーブル内フィールドの合計にも使用） |
| `IF(条件, 真, 偽)` | 条件分岐 |
| `AND(条件1, ...)` / `OR(条件1, ...)` / `NOT(条件)` | 論理演算（最大32引数） |
| `ROUND` / `ROUNDDOWN` / `ROUNDUP` | 四捨五入 / 切り捨て / 切り上げ |
| `YEN(数値, 桁数)` | 円形式表示 |
| `DATE_FORMAT(日時, フォーマット, TZ)` | 日時の表示形式変換 |
| `CONTAINS(フィールド, 値)` | チェックボックス/複数選択の選択判定 |

### 1.5 ルックアップ

- 他アプリに登録された情報を参照してデータをコピー取得
- キーに指定可能: 文字列（1行）、数値、計算、ルックアップ、リンク、レコード番号
- テーブル内フィールドはコピー先に指定不可
- 1文字検索を有効にすると部分一致検索可能

### 1.6 関連レコード一覧

- レコード詳細画面に条件一致レコードを一覧表示
- 同じアプリ/他アプリのどちらも可
- **制限**: 一覧画面には表示不可、ファイル書き出し不可、集計/検索対象外

### 1.7 数値の有効桁数

- 全体の桁数: 1〜30（小数部含む）
- 小数部の桁数: 0〜10
- 丸めかた: 最近接偶数への丸め（初期値）、切り上げ、切り捨て

## 2. 権限設定

### 2.1 アクセス権の3レベル

| レベル | 対象 |
|--------|------|
| アプリのアクセス権 | アプリ全体 |
| レコードのアクセス権 | レコード1件ずつ |
| フィールドのアクセス権 | フィールド1つずつ |

### 2.2 アプリのアクセス権（7種類）

| 権限 | 単独付与 | 依存 |
|------|----------|------|
| レコード閲覧 | 可 | なし |
| レコード追加 | 可 | なし |
| レコード編集 | 不可 | 閲覧必須 |
| レコード削除 | 不可 | 閲覧必須 |
| アプリ管理 | 可 | なし |
| ファイル読み込み | 不可 | 追加必須 |
| ファイル書き出し | 不可 | 閲覧必須 |

- **初期設定**: 作成者=全権限、Everyone=閲覧/追加/編集/削除のみ
- 対象: ユーザー/組織/グループ（組織は下位組織への継承オプションあり）
- **cybozu.com共通管理者**: 全アプリでアプリ管理権限を保有

### 2.3 レコードのアクセス権

- 設定可能権限: 閲覧/編集/削除
- 条件: フィールド値に基づく（AND/OR選択可能）
- 対象ユーザーの動的指定: ユーザー選択、組織選択、グループ選択、作成者、更新者、作業者
- **条件に指定できないフィールド**: ラベル、リッチエディター、文字列（複数行）、添付ファイル、関連レコード一覧、スペース、罫線

### 2.4 フィールドのアクセス権

- 設定可能権限: 閲覧/編集
- **設定できないフィールド**: ラベル、スペース、罫線、レコード番号、テーブル内フィールド

### 2.5 権限の優先順位

- **行の順序**: 上の行が優先（全レベル共通）
- **レベル間**: アプリ権限 > レコード権限 > フィールド権限（上位で権限がなければ下位は無関係）

### 2.6 アプリグループ

- 複数アプリに同じアクセス権を一括設定
- 初期グループ: Public（デフォルト）、Private（作成者のみ）
- スペース内アプリはPublic固定
- 変更は即時反映（アプリ更新不要）

## 3. プロセス管理

### 3.1 基本概念

| 用語 | 説明 |
|------|------|
| ステータス | レコードの処理状況（例: 申請前、申請中、承認済） |
| 作業者 | そのステータスで作業を任されているユーザー |
| アクション | ステータスを変更する操作。ボタンとして表示 |

### 3.2 ステータスの種類

- **最初のステータス**: レコード作成時に自動設定
- **途中のステータス**: 処理の中間段階
- **最後のステータス（完了）**: 作業者やアクションの設定不要

### 3.3 作業者の指定方法（3種類）

| 指定方法 | 説明 | ステータス変更条件 |
|---------|------|------------------|
| 次のユーザーから選択 | 候補から1人を前の作業者が指定 | 指定された1人がアクション実行 |
| 次のユーザー全員 | 設定ユーザー全員が作業者 | **全員**がアクション実行 |
| 次のユーザーのうち1人 | 設定ユーザー全員が作業者 | **誰か1人**がアクション実行 |

### 3.4 アクション条件

- フィールドの値に基づいてアクションボタンの表示/非表示を制御
- 例: 金額が10万円未満なら課長承認、10万円以上なら部長承認
- **条件に指定できないフィールド**: 文字列（複数行）、リッチエディター、ラベル、添付ファイル、関連レコード一覧

### 3.5 作業者以外でも実行できるアクション

- 代理承認、申請取り戻し、承認スキップに利用
- 実行可能ユーザー: すべてのユーザー or 指定したユーザー/組織/グループ

### 3.6 サンプルフロー

**差し戻し・却下あり**:
```
申請前 →[申請]→ 申請中 →[承認]→ 承認済
                        →[却下]→ 却下
                        →[差し戻し]→ 差し戻し →[再申請]→ 申請中
```

**条件分岐（金額別）**:
```
申請前 →[費用>=100万]→ 部長承認待ち →[承認]→ 承認済
       →[費用<100万]→  課長承認待ち →[承認]→ 承認済
```

### 3.7 設定変更時の注意

- 無効化しても既存レコードのステータス・履歴は保持
- 過去に削除したステータスと同じ名前は再利用不可
- 作業者設定を変更しても既存レコードの作業者は変わらない
- 利用停止ユーザーが作業者の場合、ステータス変更不可

## 4. ビュー（一覧）

### 4.1 ビューの種類

| 種類 | 説明 |
|------|------|
| 表形式 | 標準的な行×列形式。インライン編集可能 |
| カレンダー形式 | カレンダーの日付欄にレコード表示。1ページ500件まで |
| カスタマイズ | HTML記述で自由にカスタマイズ。システム管理者のみ作成可 |

### 4.2 デフォルト一覧

- **（すべて）**: 初期設定。全レコード・全フィールド表示。編集/削除不可
- 一覧タブで一番上に配置した一覧がデフォルト

### 4.3 一覧の設定

- 表示フィールドをドラッグ&ドロップで配置
- ソート: 最大5つまで指定可能
- 表示件数: 初期20件（ユーザーごとに変更可能）
- 複製・編集・削除・並び替え可能
- 表形式⇔カレンダー形式の変更は可能。カスタマイズ形式は変更不可

### 4.4 フィルター（絞り込み）

- キーワード含む/含まない、等しい、空欄判定、添付ファイル有無
- 複数条件: AND条件で組み合わせ
- **保存**: アプリ管理権限が必要。**適用**: 権限不要

### 4.5 カラム（列幅）

- フィールド境界線ドラッグで変更
- アプリ設定に保存可能（管理権限必要）
- ユーザー個別設定がアプリ設定より優先
- px単位での指定も可能

## 5. レポート（グラフ）

### 5.1 グラフの種類

- 横棒/縦棒グラフ（集合/積み上げ/100%積み上げ）
- 折れ線グラフ / 曲線グラフ
- 面グラフ / 曲線面グラフ（積み上げなし/積み上げ/100%積み上げ）
- 円グラフ
- 表 / クロス集計表

### 5.2 集計設定

- 分類: 大項目/中項目/小項目の最大3階層
- 集計方法: レコード数、合計、平均、最大値、最小値
- ソート: 集計値 or 分類項目のフィールド値で昇順/降順
- 1アプリ最大1,000件のグラフ保存可能
- 集計可能レコード数: 先頭から100万件まで

### 5.3 定期レポート

- 集計間隔: 1時間/毎日/毎週/毎月/毎四半期/毎年
- 過去30回分の集計結果を保持
- 設定後はグラフの編集・複製不可

### 5.4 集計結果の活用

- CSV/Excel書き出し（ファイル書き出し権限必要）
- 埋め込み用HTMLタグ取得可能
- グラフ上の集計結果クリックで該当レコードに絞り込み

## 6. 管理者設定

### 6.1 管理者の種類（6種類）

| 管理者 | 権限範囲 |
|--------|----------|
| cybozu.com共通管理者 | 最上位。全アプリ設定変更、ユーザー/組織管理、監査ログ |
| 組織の管理者 | 特定組織のメンバー管理 |
| kintoneシステム管理者 | kintone全体設定（アプリ/スペース管理、テンプレート、プラグイン） |
| アプリ管理者 | 個別アプリの設定変更 |
| スペースの管理者 | 個別スペースの設定管理 |
| 契約情報の管理者 | 契約に関する管理 |

### 6.2 システム管理で設定できるアクセス権

- システムの管理権限
- アプリグループの管理権限
- アプリの作成/管理権限
- スペース/ゲストスペースの作成権限
- **上の行が優先**、`Everyone`は最下部固定

### 6.3 利用する機能の選択

通知のメール送信、スペース機能、ゲストスペース機能、ピープル・メッセージ機能等を有効/無効化可能

### 6.4 監査ログ

- cybozu.com共通管理者のみ閲覧可能
- 記録内容: 誰が/いつ/どの操作を行ったか
- 絞り込み、メール通知、ダウンロード可能

## 7. 通知

### 7.1 通知の種類

| 種類 | 説明 |
|------|------|
| 「自分宛」 | メンション、作業者変更、リマインダー等。アイコンに件数表示 |
| 「すべて」 | フォロー中の書き込み、いいね等。自分宛も含む |

### 7.2 アプリの通知設定（3パターン）

| 設定 | 説明 |
|------|------|
| アプリの条件通知 | レコード追加/編集/コメント/ステータス変更時に通知 |
| レコードの条件通知 | フィールド値が条件を満たした時に「自分宛」通知 |
| リマインダーの条件通知 | 日付フィールド基準の通知（~日前/~日後、最大10件） |

### 7.3 メール通知

- システム管理者が有効/無効を管理
- 受信設定: 「自分宛のみ」「すべて」「受信しない」
- HTML/テキスト形式選択可能

### 7.4 通知の管理

- 閲覧期間: 送信後70日間（「あとで読む」で延長）
- 絞り込み: 場所、送信者で絞り込み可能
- 一括既読 / 既読→未読の切り替え可能

## 8. レコード操作

### 8.1 追加・編集・削除

- **追加**: 一覧画面の「+」アイコン or ファイル読み込みで一括登録
- **編集**: 詳細画面 or 一覧画面でインライン編集
- **同時編集制限**: 同じレコードの同時編集不可（先に保存した操作が優先）
- **削除**: 個別削除 or 一括削除（最大5万件、管理権限+高度な設定で有効化必要）
- **削除したレコードは復元不可**

### 8.2 インライン編集

- 編集可能: 文字列、数値、選択肢系、日時系、リンク、ユーザー選択系
- 編集不可: 関連レコード一覧の条件フィールド、ルックアップのコピー先、テーブル内
- 高度な設定で無効化可能

### 8.3 レコードの再利用（コピー）

- **コピーされないもの**: レコード番号、作成者/更新者、作成日時/更新日時、添付ファイル、プロセス管理情報、コメント、変更履歴

## 9. データ管理

### 9.1 インポート（ファイル読み込み）

| 形式 | 制限 |
|------|------|
| Excel (.xlsx) | 最大1MB、1,000行 |
| CSV/TXT/TSV | 最大100MB、10万行 |

- 更新キーで一致するレコードを更新、なければ新規登録
- 更新キー: レコード番号 or 重複禁止設定の文字列（1行）
- 権限: 「ファイル読み込み」権限が必要
- 読み込めないもの: ラベル、計算、添付ファイル、関連レコード一覧、ルックアップコピー先、コメント、変更履歴、プロセス管理情報

### 9.2 エクスポート（ファイル書き出し）

- CSV形式で書き出し
- 「ファイル書き出し」権限が必要
- 先頭行を項目名にする、文字コード、区切り文字を指定可能
- テーブル含む場合は複数行（先頭列の「*」が新レコードの目印）

### 9.3 バックアップ

- ファイル書き出しでバックアップ（アプリ単位）
- 削除したアプリ/スペースは14日以内に復旧可能（共通管理者）

## 10. スペース

### 10.1 概要

チーム単位でコミュニケーションやアプリを集約する機能。

### 10.2 種類

| 種類 | 説明 |
|------|------|
| 通常スペース | 社内ユーザーのみ |
| ゲストスペース | 社外ユーザーも参加可能 |
| 複数スレッドスペース | ポータル＋複数スレッド |
| 単一スレッドスペース | 1スレッドのみ |

### 10.3 主な機能

- **スレッド**: 話題ごとにコミュニケーション分離。コメント、宛先指定、ファイル添付対応
- **スペース内アプリ**: スペースに追加したアプリ。スペース削除でもアプリは残る
- **メンバー管理**: ユーザー/組織/グループ単位で追加。公開スペースは参加リクエスト可能
- **お知らせ編集**: レコード一覧やグラフの貼り付け可能
- **テンプレート**: スペースの構成をテンプレート化。エクスポート/インポート可能

### 10.4 削除と復元

- 削除から14日以内に復元可能
- スペース内アプリは削除されない

### 11.2 管理

- 2枚目以降のポータル作成可能（ライトコース除く）

### 11.3 ウィジェット（2枚目以降）

リッチテキスト、アプリ指定、スペース指定、レコード一覧、グラフ、アプリ一覧、スペース一覧、未処理レコード、通知一覧

### 11.4 カスタマイズ

- JavaScript/CSSでカスタマイズ可能
- kintoneシステム管理画面からファイル適用

## 12. アプリ作成・管理

### 12.1 アプリ作成方法

1. サンプルアプリから追加
2. はじめから作成（D&D）
3. Excel/CSV読み込みで作成
4. テンプレートファイル読み込み
5. 既存アプリをコピー
6. 登録済みテンプレートから作成

### 12.2 アプリ管理機能

- **プレビュー（動作テスト）**: 本番と独立した環境でテスト
- **メンテナンスモード**: 利用を一時制限（API含む）
- **設定変更履歴**: いつ/誰が/どの設定を変更したか
- **言語ごとの名称**: 日本語/英語/中国語（簡体字）で自動切替
- **レコードタイトル**: 通知や検索結果に表示される名称
- **アプリコード**: 一意のコード（例: APP-001）
- **所属スペース変更**: 別スペースへの移動可能

### 12.3 テンプレート

- アプリ設定をテンプレートとして保存・再利用
- エクスポート（zip）/ インポート可能
- **含まれないもの**: レコードデータ、通知設定、APIトークン、Webhook設定、プラグイン設定値

---

## 13. JavaScriptカスタマイズ

### 13.1 kintone全体のカスタマイズ

- kintoneシステム管理画面から適用
- PC用/スマートフォン用を分けて設定
- ライトコースでは利用不可

### 13.2 アプリ個別のカスタマイズ

- アプリ設定画面から適用
- 読み込み順: kintone全体 → アプリ個別
- JavaScript/CSSそれぞれ30個まで
- URL指定 or アップロード

### 13.3 適用範囲（3段階）

1. すべてのユーザーに適用
2. 管理者だけに適用
3. 適用しない

## 14. アプリアクション

- あるアプリのレコードの値を別アプリのレコードにコピー（転記）
- レコード詳細画面にアクションボタンとして表示
- コピー元/先でフィールドタイプが一致必要
- テーブル内はテーブル同士でのみコピー可能
- 表示条件（特定ステータスのみ等）設定可能

---

## 15. カテゴリー

- レコードをカテゴリーで分類管理
- 一覧画面の左側にカテゴリーツリー表示
- 最大3階層の親子構造
- 対応フィールド: ドロップダウン、ラジオボタン、チェックボックス、複数選択、文字列（1行）
- 無効化しても設定は保持（再有効化で復元）

---

## 16. コメント

- レコード詳細画面でコメント投稿・閲覧
- 宛先指定（@メンション）、返信可能
- 編集/削除: 投稿者本人 or アプリ管理者
- いいね機能あり
- 無効化可能（既存コメントは保持）

---

## 17. Webhook

- 特定操作時に外部サービスへ自動送信
- トリガー: レコード追加/編集/コメント/ステータス変更/削除
- 1アプリ最大10個
- HTTPS通信のみ
- ベストエフォート（送信保証なし）
- 送信ログで成功/失敗確認可能
- ライトコースでは利用不可

---

## 18. APIトークン

- 外部プログラムからのアクセス認証用トークン
- 設定可能権限: レコード閲覧/追加/編集/削除、アプリ管理
- 1アプリ最大20個
- トークンごとに権限を個別設定
- 再生成（無効化→新トークン生成）可能
